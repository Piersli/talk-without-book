# YouMind Skill: 读后无书 · 内容拆解

> YouMind 上的"拆书生成器"版本。与本地版相比：不读本地文件，不写本地知识库，**生成的内容输出为 YouMind document，需要用户/Hermes 后续同步回本地**。

## skill 元数据

```yaml
name: 读后无书内容拆解
display_name: 读后无书 · 内容拆解
description: 把一本书或一篇文章按"道法术器势"五维框架拆解，输出结构化节点文档。适合 board 里有书籍材料的场景。
tags: [reading, extraction, books, dao-fa-shu-qi-shi]
category: 知识 / 内容生产
visibility: private
```

## instructions（10 步）

1. **确认输入**：用户应该上传过一本书/文章作为 material，或者在指令里给出材料 ID。如果没有，问用户："请先用 `createMaterialByUrl` 或上传文件，然后告诉我 material ID。"

2. **读取框架**：找到 board 里名为「framework.md」的 material，读取它——这是道法术器势的判定规则。如果找不到，使用内置默认：道=永恒真相、法=因果规律、术=可行动方法、器=工具模板、势=条件趋势。

3. **读取当前道库**：找 board 里名为「道.md」的 material，读取现有道索引（如果存在）。如果不存在或为空，标记"这是第一本书，将主动建议初始道集"。

4. **读取目标书**：读取用户指定的书/文章 material 的完整内容。如果超过 ~50000 字，按章节自然分块。

5. **Phase 1 · 全书概要**：对每一章生成：章节名称、核心论点（1-3 句）、关键概念列表、重要研究/案例/人物、与每条现有道的关联度（0-3 分）。汇总成全书概要。识别反复出现但与现有道关联弱的主题（候选新道）。

6. **Phase 2 · 法的提取**：每条法必须包含：(a) A → B 的因果机制；(b) 原文依据（章节 + 论据）；(c) 适用边界；(d) 上位道编号。**法不能与上位道同义复述**——必须增加道所没有的具体因果机制。预计 25-45 条。

7. **Phase 3 · 术 / 器 / 势的提取**：
   - 术：具体可行动方法，标注上位法（描述性引用）。预计 15-30 条。
   - 器：工具、模板、案例、分类框架。预计 10-25 条。
   - 势：条件、边界、趋势判断，标注时效性和影响范围。预计 5-15 条。

8. **质量自检**：对照清单——每条法都有上位道？每条术都可直接行动？每条器都可直接复用？每条势都标了时效？无跨维度同义重复？候选新道是否真的没被现有道覆盖？

9. **生成五份 document**：调用 `createDocumentByMarkdown` 5 次，分别创建：
   - 《{书名}》· 法
   - 《{书名}》· 术
   - 《{书名}》· 器
   - 《{书名}》· 势
   - 《{书名}》· meta（含触碰的道、节点统计、跨书连接、候选新道）

10. **建议下一步**：告诉用户:
    > ✅ 拆完了。五份文档已在 board。
    >
    > 候选新道：{N} 个（如有，列出标题和理由）
    >
    > 下一步：
    > - 如果你想把这些并入本地 `$TWB_ROOT/books/`：在 Hermes 里说"把 YouMind 上《{书名}》的拆解同步进来"
    > - 如果想用 `读后无书结构审视` 评审候选道：在本 board 里调用那个 skill

## 关键约束

- **候选新道必须人工确认**。不要在文档里直接把新道当作正式道——要标注"候选道 A / B / C"。
- **不要写入本地文件**。YouMind 没那个能力。只生成 document，让 Hermes 后续同步。
- **法必须有具体因果**。同义复述是最常见的低质量错误。
- **中文输出**。所有节点内容、标题、描述用中文。
