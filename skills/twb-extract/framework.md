# 道法术器势 — 五维度信息拆解分级规则

> 本文件是 **读后无书（talk without book）** 项目的核心分类规范。所有书籍信息节点的提取、归类、去重都必须严格遵守此规则。

---

## 一、总体架构

```
道（为什么世界是这样的）
 └─ 法（在这个世界里，什么规律可以依赖）
     └─ 术（依据这些规律，具体怎么做）
         └─ 器（做的时候，用什么工具/模板/案例）
势（以上四层在什么条件下成立，风往哪吹）
```

五个维度是**层级关系**，不是并列分类。道→法→术→器是从抽象到具体的推导链；势是横切所有层的**边界条件**。

---

## 二、各维度严格定义与判定标准

### 道 — 永恒不变的底层真相

**定义**：关于人性、物理世界、复杂系统的第一性原理和永恒规律。不依赖于任何一本书、任何一个作者——书只是触碰到了它们，不是发明了它们。道是跨时代、跨文化、跨领域成立的根本性事实。

**道的本质特征**：
- **永恒性**：千年前成立，千年后依然成立
- **普遍性**：不限于某个领域、某个文化、某个时代
- **不可证伪性**：不是"在某些条件下成立"的规律（那是法），而是无条件的底层事实
- **全局共享**：道不属于某本书，整个知识库的道是统一的，多本书共享同一套道

**判定测试**：
1. **永恒测试**：这条在1000年前成立吗？1000年后还会成立吗？ → 不确定，降级为法或势
2. **跨域测试**：这条是否在完全不同的领域（物理、生物、经济、心理）都能找到对应？ → 能，才是道
3. **第一性测试**：这条能否从更基本的原理推导出来？ → 能，则它不是道，是法；道是推导链的起点
4. **坍塌测试**：如果这条不成立，知识库中多少条法会失去根基？ → 影响面广才是道

**道的类别**（非穷举）：
- **人性类**：关于人类认知、情感、行为的永恒模式（如：人类认知与世界复杂度存在结构性错配）
- **世界类**：关于物理世界和复杂系统的基本属性（如：世界是因果高度耦合的复杂系统）
- **关系类**：关于事物之间永恒不变的结构性关系（如：代价与价值不可分离）

**数量约束**：整个知识库的道预计在 **20-50 条**量级（不是每本书3-8条，而是全局总数有限）。一本书通常触碰其中 3-8 条。

**数据结构**：道不存储在 `books/{Book}/道.md` 中，而是存储在全局的 `dao/` 目录下。每本书的提取结果中标注"本书触碰了哪些道"。

**道的节点格式**：
```markdown
## [永恒真相 — 一句断言]

- **类别**：人性类 / 世界类 / 关系类
- **表述**：用一段话展开这条道的含义
- **触碰此道的书籍**：哪些书触碰了这条道，各自从什么角度
- **下位法**：知识库中哪些法是从这条道推导出来的
- **与其他道的关系**：逻辑上的互补、互为条件、共同构成某个更大图景
```

---

### 法 — 可依赖的因果规律

**定义**：从道衍生出的结构化规则，是作者明确论证的因果关系、原则、框架。回答"依据什么规律"和"为什么这样做有效"。

**判定测试**：
1. **因果测试**：这条是否描述了一个 A→B 的因果/相关关系？ → 不是因果，可能是道或术
2. **可证伪测试**：这条在什么条件下会失效？ → 如果找不到失效条件，它可能是道（无条件成立的信念）
3. **推导测试**：这条能否从某个道推导出来？ → 能，则它是法；不能，则它可能本身是道
4. **去重规则**：如果一条法只是某个道的换一种说法，删除它；法必须在道的基础上**增加了信息**（通常是增加了具体的因果机制或适用边界）

**法的节点格式**：
```markdown
## [规律名称 — 一句因果表述]

- **因果机制**：A 如何导致 B
- **原文依据**：作者的论证和案例
- **适用边界**：在什么条件下成立，什么条件下失效
- **上位道**：这条法从哪个道推导而来
```

---

### 术 — 可执行的方法策略

**定义**：基于法的操作性建议。回答"具体怎么做"。

**判定测试**：
1. **行动测试**：读完这条，是否可以直接采取行动？ → 不能直接行动，降级为法
2. **依据测试**：这条方法背后依据的是哪条法？ → 找不到对应的法，说明缺少中间层，需要补法
3. **场景测试**：这条方法适用于什么具体场景？ → 说不清场景，可能太抽象，属于法

**术的节点格式**：
```markdown
## [方法名称 — 动词开头的一句话]

- **方法描述**：具体怎么做
- **原文依据**：来自哪个章节/谁说的
- **适用场景**：什么时候用
- **上位法**：这条术依据的是哪条法
```

---

### 器 — 工具、框架、案例、模板

**定义**：执行术时使用的具体工具、分类体系、度量方式、可直接复用的案例模板。

**判定测试**：
1. **复用测试**：这个东西能否被直接拿来用（填模板、套框架、引用案例）？ → 不能直接复用，可能是术
2. **具体性测试**：是否有明确的结构、步骤、分类项？ → 没有，可能是术

**器的节点格式**：
```markdown
## [工具/框架名称]

- **描述**：这是什么，怎么用
- **原文出处**：来自哪里
- **使用方式**：在什么情况下拿出来用
- **上位术**：配合哪条术使用
```

---

### 势 — 条件、边界、趋势

**定义**：以上四层成立的时代背景、条件依赖、趋势判断。势是唯一一个**横切所有层级**的维度。

**判定测试**：
1. **条件测试**：这条是否描述了某个道/法/术/器成立的前提条件或时代背景？ → 是，归入势
2. **方向测试**：这条是否描述了某种正在发生的结构性趋势？ → 是，归入势
3. **时效测试**：这条判断是否会随时间失效？ → 如果永远成立，它可能是道或法，不是势

**势的节点格式**：
```markdown
## [趋势/条件判断 — 一句话]

- **判断内容**：什么正在发生/什么条件需要满足
- **原文依据**：作者的论据
- **影响范围**：这个势影响了哪些道/法/术/器的适用性
- **时效性**：这个判断预计在什么时间尺度内有效
```

---

## 三、跨层级关系规则

1. **每个法必须指向至少一个道**（上位关系）
2. **每个术必须指向至少一个法**（上位关系）
3. **每个器应指向至少一个术**（配合关系）
4. **势可以关联任何层级的节点**（横切关系）
5. **同一内容不得出现在两个维度中** — 如果出现重复，按"最高适用层级"归入（道 > 法 > 术 > 器）
6. **关联节点用 [[节点标题]] 语法标注**，为后续 Wiki 连接做准备

## 四、数据结构

```
talkwithbook/
  dao/                    # 全局道（跨书共享）
    道.md                 # 全部道节点，持续积累
  books/
    {Book_Name}/
      法.md               # 本书的法（标注上位道编号）
      术.md               # 本书的术（标注上位法）
      器.md               # 本书的器（标注上位术）
      势.md               # 本书的势（横切所有层）
      meta.md             # 本书触碰了哪些道，以及从什么角度
```

关键区别：**道是全局的，法/术/器/势是 per-book 的**。当多本书的法指向同一条道时，跨书连接自然形成。

## 五、质量检查清单

- [ ] 道通过了永恒测试（千年尺度成立）
- [ ] 道通过了跨域测试（多领域可找到对应）
- [ ] 道通过了第一性测试（不能从更基本的原理推导）
- [ ] 一本书触碰的道在 3-8 条之间
- [ ] 法没有与道重复（每条法在道的基础上增加了因果机制信息）
- [ ] 每条法标注了上位道
- [ ] 术都是可直接行动的
- [ ] 器都是可直接复用的
- [ ] 势标注了时效性
- [ ] 无跨维度重复
